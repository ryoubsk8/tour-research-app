# Performance Test Instructions — 旅スポ (TABISPO)

## ステータス: 簡易（数値目標なし）
NFR要件（Q4=A）により、本MVPでは**厳密な数値目標は設定しない**。体感の軽快さを目視で確認する。

## 確認観点
- 起動 → 一覧表示までが体感で素早いか（各カテゴリ約10件に制限済み）
- タブ切替が即時か（起動時取得の結果を保持・再利用しているため再検索が走らない）
- 画像読み込みで画面がもたつかないか（`loading="lazy"`・サムネサイズ指定済み）

## 簡易な計測方法（任意）
- ブラウザ DevTools の Network / Performance タブで、初回表示までの時間やAPI呼び出し回数を確認
- Lighthouse（DevTools）でモバイル想定のスコアを参考値として取得

## 将来の最適化候補（MVP対象外）
- 検索結果のキャッシュ、検索タイミングの調整、画像の最適化
- Google Places 呼び出し回数の削減（コスト最適化）
