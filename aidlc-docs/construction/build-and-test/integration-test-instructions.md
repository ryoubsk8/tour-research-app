# Integration Test Instructions — 旅スポ (TABISPO)

## Purpose
単一ユニットのフロントエンドアプリのため、ユニット間の自動結合テストは限定的。
ここでは「アプリ ↔ 外部サービス（Google Places / ブラウザ位置情報）」の結合を、実際のAPIキーを使った**手動スモークテスト**で確認する。

## 前提
- `.env` に有効な `VITE_GOOGLE_MAPS_API_KEY` を設定済み（Maps JavaScript / Places / Geocoding を有効化、キー制限済み）
- `npm install` 済み

## 手順
```bash
cd tour-research-app
npm run dev
# 表示されたURL（通常 http://localhost:5173）をブラウザで開く
```

## 手動テストシナリオ（要件対応）
| # | シナリオ | 期待結果 | 要件 |
|---|---|---|---|
| 1 | 起動して位置情報を「許可」 | 現在地周辺の観光スポットが距離が近い順に表示され、初期タブは「観光」 | FR-1, FR-3, FR-6 |
| 2 | 「グルメ」タブに切替 | グルメの一覧が表示される（起動時に取得済みで即表示） | FR-3, FR-4 |
| 3 | 細分類（例: カフェ）を選択 | 該当ジャンルに絞り込まれて再検索される | FR-5 |
| 4 | 検索範囲を切替（例: 約1km→約10km） | 新しい範囲で再検索され件数/内容が変わる | FR-9 |
| 5 | 「営業中のみ」をON | 営業中のスポットに絞り込まれる | FR-8 |
| 6 | スポットカードの「営業時間を見る」 | 曜日別の営業時間（取得できる場合）が展開表示される | FR-7 |
| 7 | 「Googleマップでナビ」 | 別タブでGoogleマップの経路案内が開く | FR-10 |
| 8 | ☆を押す→「お気に入り」タブ | 保存され、お気に入り一覧に表示。再読み込み後も保持 | FR-11 |
| 9 | 位置情報を「ブロック」して再読み込み | 地名入力欄が表示され、地名（例: 京都駅）で検索できる | FR-2 |
| 10 | オフライン/無効キーで検索 | エラーメッセージ＋「再試行」ボタンが表示される | FR-12 |

## 注意
- Google Places の結果はリアルタイムのため、件数・内容は実行時の状況により変動する。
- 自動E2E（Playwright等）は本MVPのスコープ外。将来の拡張候補。
